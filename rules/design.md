## 2. 我们对“好的个人网站”的定义

好的个人网站不是“炫技堆砌”，而是：

1. **3 秒内说明你是谁、做什么、能带来什么价值**  
   首屏必须清晰：身份 + 能力 + 行动入口（联系/作品）。

2. **信息结构清楚，用户不需要猜**  
   常见最短路径：About -> Work/Projects -> Contact。

3. **作品有上下文，不只是截图**  
   每个项目至少回答：问题、你的贡献、结果（指标或影响）。

4. **视觉有识别度，但不牺牲可读性和效率**  
   风格服务叙事，不压过内容。

5. **可访问、响应式、加载稳健**  
   手机可读、键盘可用、对比度达标、交互反馈明确。

## 3. 关键设计原则（从 skills 与案例抽取）

### 3.1 叙事优先
1. 首屏一句话价值主张，避免空泛形容词。
2. “我会什么”要转成“我如何解决问题”。
3. CTA 不超过两个：`查看项目`、`联系入口`。

### 3.2 信息架构优先
1. 页面建议结构：
   - Hero
   - About（短）
   - Selected Projects（重点）
   - Experience/Capabilities（可折叠）
   - Contact（收口）
2. 导航标签用用户语言，不用内部术语。

### 3.3 视觉与排版
1. 正文行长控制在约 65-75 字符。
2. 标题层级要有明显比例差，不做“同号字假层级”。
3. 节奏通过留白变化建立，不要所有区块同间距。
4. 尽量减少“卡片套卡片”。

### 3.4 色彩与品牌
1. 先定色彩策略，再选颜色（克制/强调/全调色盘）。
2. 避免默认 AI 味：
   - 紫蓝渐变泛滥
   - 过度玻璃拟态
   - 无意义渐变文字
3. 色彩应增强层级与引导，而非纯装饰。

### 3.5 交互与动效
1. 动效有功能目的：引导、反馈、转场。
2. 避免弹簧乱跳，优先平滑 ease-out。
3. 提供明确状态：hover、focus、loading、error、success。

### 3.6 内容质量
1. 文案短句、可扫描、可证据化。
2. 项目描述优先“结果导向”（指标、影响、学习）。
3. 联系方式要低摩擦（邮件/社媒一跳可达）。

## 4. 个人网站评估清单（发布前自检）

总分 100，建议 >= 80 才发布：

1. 定位清晰（15）
   - 首屏是否在 3 秒内可理解。
2. 项目说服力（20）
   - 是否有问题-行动-结果闭环。
3. 信息架构（15）
   - 导航与页面流是否自然。
4. 可读性与排版（15）
   - 层级、行长、留白、对比度是否合格。
5. 视觉识别（10）
   - 是否有个人风格但不过度装饰。
6. 交互反馈（10）
   - 状态反馈是否完整。
7. 响应式与可访问性（10）
   - 手机端、键盘、语义与对比度。
8. 性能与稳定（5）
   - 首屏加载、图片策略、无明显卡顿。

## 5. 常见反模式（明确避免）

1. 首屏只有大图和空话，没有价值主张。
2. 作品只有视觉，没有背景与结果。
3. 全站“同一种卡片”重复，缺少层次。
4. 颜色很花，但文字对比不足。
5. 动效过多导致眩晕或分心。
6. 移动端排版崩溃，按钮难点击。
7. 联系入口藏太深，转化路径中断。

## 6. 可直接落地的页面骨架（推荐）

### 首页
1. Hero
   - Name / Role / One-line Value
   - CTA1: 查看项目
   - CTA2: 联系入口
2. Selected Work（3-5 个）
   - 项目名 + 一句话成果 + 技术标签
3. About（短）
   - 关注方向、工作方式、偏好场景
4. Contact
   - 邮箱 + 社媒 + 简历链接

### 项目详情页模板
1. 背景与目标
2. 我的职责
3. 关键决策（设计/技术）
4. 结果（指标/业务影响）
5. 复盘（做对了什么、下次会改什么）

## 7. 后续执行建议（用于 AI 协作）

1. 先定义“目标受众 + 目标机会”再生成页面。
2. 先生成结构稿，再做视觉稿，再做动效稿。
3. 每轮改版只改一个维度（结构/视觉/文案/性能）并评分。
4. 用上面的 100 分表做客观收敛，避免“越改越花”。

## 8. DurianStudio 首版实现约定

### 8.1 项目定位

DurianStudio 是个人中心网站，不是博客系统。它集中呈现：

1. 个人介绍与联系方式。
2. 精选作品与实验入口。
3. 外部博客、文章、笔记链接。
4. 后续可替换为真实项目与真实链接的高质量占位内容。

### 8.2 技术与环境

1. 框架：Astro。
2. 包管理器：pnpm。
3. 样式：Tailwind CSS v4 入口保留在 `src/styles/global.css`，首版以自定义 CSS 变量和全局样式实现完整视觉系统。
4. 图标：lucide-astro。
5. 动效：首版使用轻量 CSS 动效，不引入复杂交互运行时。
6. 部署：Cloudflare Pages 暂缓，先把网站效果做好。
7. 结构约定：页面结构保留在 `src/pages/index.astro`，客户端交互脚本独立放在 `public/scripts/site.js`，避免把大段 JavaScript 写在 Astro 页面里。

### 8.3 首版页面结构

1. 顶部导航：DurianStudio / 首页 / 朕の天下 / 联系，并在导航右侧提供浅色/黑色主题切换按钮。
2. Hero：背景图、头像、高级质感标题 `Welcome to DurianStudio`，标题使用 `assets/zhea.woff2`，仅添加彩色渐变文字效果。
3. 删除大标题 `Durian Studio` 和介绍段落，不再展示长文字简介。
4. Links 模块合并到头像下方联系方式区域：联系方式图标一行展示，点击 GitHub / Blog / Email / Chat 后在同一区域展示对应信息。
5. 桌面端个人名片整体靠左，右侧保留内容板块；移动端改为单列居中。
6. 右侧添加“今日小报告”板块：请求浏览器定位权限，通过 Open-Meteo 获取简略天气；定位失败或请求失败时显示“阳光明媚，云朵正在摸鱼。”。
7. 右侧添加“朕の天下”项目板块，用于显示项目入口；首版只保留 Blog，后续再追加项目。
8. 不做卡片叠卡片，右侧项目列表使用横向信息行、分隔线和图标，不在项目外再套小卡片。
9. 首屏下方先保留渐变尾巴，并在过渡空场中放置大号欢迎语 `欢迎光临 (っ˘꒳˘ς)`；再进入中间像素墙展示区。像素墙位置不因欢迎语上提，中间区只放 `assets/wall/light.html` 与 `assets/wall/dark.html` 转换出的两套墙，不加多余解释文字。
10. 底部保留简洁“关于 DurianStudio”footer，用于站点说明、社交入口和基础链接。
11. 主题切换使用 `data-theme` + `localStorage` 保存选择；默认跟随系统偏好，无法读取时回落黑色主题。

### 8.4 视觉方向

1. 方向：明亮、轻快、有科技感，但避免中二表达。
2. 参考：`zhea.vip` 的年轻感、圆润卡片、头像/个人站亲和力，但不复刻其博客结构。
3. 色彩：蓝绿、青色、荧光绿、橙色点缀，允许霓虹光效，但要保持信息可读。
4. 首页必须包含：背景图、头像、个人简介、联系方式、顶部菜单栏。
5. 首页下方的像素墙是装饰展示，不是主题说明区；不要在像素墙旁添加解释性标题、长文案、Blog 链接或作品堆叠。
6. wallpaper 在浅色主题中尽量保持原图直出，不叠加白色遮罩；黑色主题只允许极轻低亮度覆盖，避免背景被闷住。
7. 黑色主题不在首页四周叠加暗角或径向压暗，只允许使用很低透明度的均匀覆盖和底部自然过渡，避免“黑框感”。
8. 浅色主题保留明亮轻快感，黑色主题保留科技感与霓虹点缀，两套主题共享布局与动效。
9. 联系方式按钮使用方形圆角图标按钮，不使用纯圆形按钮。
10. 上方 Hero 与下方渐变尾巴之间必须有明显柔和渐变衔接，不允许硬切或色块压住主页卡片。
11. 资源约定：
   - 网站图标：`assets/icon.svg`
   - 头像：原 `assets/头像.png` 已改名为 `assets/avatar.png`，避免中文路径构建/部署问题。
   - 背景图：`assets/wallpaper.png`
   - 像素墙参考稿：`assets/wall/light.html`、`assets/wall/dark.html`；当前已转换为 Astro 页面里的 Canvas 版本。

### 8.7 性能约定

1. 像素墙不使用 iframe，不嵌套完整 HTML，也不使用 896 个 DOM 像素格逐个改 style。
2. 像素墙使用 `canvas` 绘制：每套主题一个 canvas，dark/light 两套仍保留在页面结构中。
3. 当前主题、进入视口、页面可见，三者同时满足时才开启动画；隐藏主题和离屏状态必须暂停。
4. 空白贡献墙预渲染到离屏 canvas，动画帧只绘制发光像素。
5. 帧率限制在约 24fps，`devicePixelRatio` 绘制上限为 1.5，避免高分屏造成无意义性能开销。
6. 中间展示区使用 `content-visibility: auto` 和 `contain` 隔离布局与绘制成本。

### 8.6 参考检索结论

1. 背景图与下一段衔接：使用同色系底色、底部渐变淡出、延续轻网格纹理，避免直接硬切。
2. 下方主题选择：浅色主题更适合当前 wallpaper 的绿色/浅色调；深色用于深绿/深青过渡，不在首页四周制造暗角。
3. 设计库参考：Lapa Ninja、Landing Gallery、Refto、Collected 等站点都强调“一个明确机制 + 清楚信息层级”。DurianStudio 当前先稳定 Hero 与渐变衔接，再继续扩展下方视觉机制。
4. Chrome 官网参考：大标题、短说明、模块化功能卡片，适合用于底部站点简介区。

### 8.5 首版验收

1. `pnpm astro check` 通过。
2. `pnpm build` 通过并生成 `dist/`。
3. `pnpm preview` 可正常预览。
4. 桌面端首屏能看到 Hero 底部渐变提示。
5. 移动端无明显文本溢出、按钮重叠或导航不可用。
